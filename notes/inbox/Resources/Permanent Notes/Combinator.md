# Combinator

A combinator is any function which only depends on its inputs[^1]

[^1]: Wlashin, Scott. “Understanding Parser Combinators: A Deep Dive.” Presented at the NDC 2016, Oslo, Norway, August 8, 2016. <https://youtu.be/RDalzi7mhdY?t=833>

Often used as shorthand for "Combinator Pattern" or "Combinator library".
Combinator libraries are libraries of functions that take multiple things of the same type as input, and produce a thing of that same type as output[^2].

[^2]: https://wiki.haskell.org/Combinator
